# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions CI (build + test on every branch) and Docker image build + push to ghcr.io on main, with path-filtered matrix so only changed services rebuild.

**Architecture:** Two workflow files: `ci.yml` gates PRs and branches; `docker.yml` runs on main only, using `dorny/paths-filter` to detect changes and a matrix strategy to build only affected service images. All 7 services get their own Dockerfile following a shared multi-stage pattern with SharedKernel copied for layer-cache efficiency.

**Tech Stack:** GitHub Actions, Docker Buildx, ghcr.io, `dorny/paths-filter@v3`, `docker/metadata-action@v5`, `docker/build-push-action@v5`, `dorny/test-reporter@v1`

## Global Constraints

- All Dockerfiles use `mcr.microsoft.com/dotnet/sdk:8.0` build stage and `mcr.microsoft.com/dotnet/aspnet:8.0` final stage — no other base images
- Build context for ALL Dockerfiles is the repo root (`.`) — never the service subdirectory
- SharedKernel must be copied before service source in every Dockerfile — preserves layer caching
- `TreatWarningsAsErrors=true` is already enforced by `Directory.Build.props` — build fails on any warning; do not add `--warnaserror` flag (redundant)
- No package versions in individual csproj files — centrally managed in `Directory.Packages.props`
- Image names: `ghcr.io/<owner>/tcgtrading-<service-name>` — all lowercase, hyphen-separated
- `docker.yml` requires `permissions: packages: write` — without this ghcr.io push returns 403
- `GITHUB_TOKEN` is used for ghcr.io auth — no additional secrets needed
- Matrix service names must exactly match `dorny/paths-filter` filter keys — used for dynamic output indexing

Full replacement for `src/IdentityService/Dockerfile`:

```dockerfile
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src

# Layer 1: Central build config — changes rarely, cached longest
COPY Directory.Build.props Directory.Packages.props ./

# Layer 2: Project files only — restore cached until deps change
COPY src/SharedKernel/TCGTrading.SharedKernel.csproj src/SharedKernel/
COPY src/IdentityService/TCGTrading.IdentityService.csproj src/IdentityService/

RUN dotnet restore src/IdentityService/TCGTrading.IdentityService.csproj

# Layer 3: Full source — invalidates restore cache only on source change
COPY src/SharedKernel/ src/SharedKernel/
COPY src/IdentityService/ src/IdentityService/

RUN dotnet publish src/IdentityService/TCGTrading.IdentityService.csproj \
    -c Release -o /app/publish

FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS final
WORKDIR /app
# curl required for Docker healthcheck in docker-compose.yml
RUN apt-get update && apt-get install -y --no-install-recommends curl \
    && rm -rf /var/lib/apt/lists/*
COPY --from=build /app/publish .
ENTRYPOINT ["dotnet", "TCGTrading.IdentityService.dll"]
```

### Task 3: Create `ci.yml` — build and test workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Produces: Workflow that runs on every push and PR, builds the solution, runs all 35 tests

- [ ] **Step 1: Create `.github/workflows/` directory and `ci.yml`**

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: ["**"]
  pull_request:
    branches: [main]

permissions:
  contents: read
  checks: write   # required for dorny/test-reporter to post check results

jobs:
  build-and-test:
    name: Build & Test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup .NET 8
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: 8.0.x

      - name: Cache NuGet packages
        uses: actions/cache@v4
        with:
          path: ~/.nuget/packages
          key: ${{ runner.os }}-nuget-${{ hashFiles('**/Directory.Packages.props') }}
          restore-keys: |
            ${{ runner.os }}-nuget-

      - name: Restore
        run: dotnet restore TCGTrading.sln

      - name: Build
        run: dotnet build TCGTrading.sln --no-restore -c Release

      - name: Test
        run: dotnet test TCGTrading.sln --no-build -c Release --logger "trx;LogFileName=results.trx" --results-directory TestResults

      - name: Test report
        uses: dorny/test-reporter@v1
        if: always()
        with:
          name: .NET Tests
          path: TestResults/*.trx
          reporter: dotnet-trx
```

Note: `TreatWarningsAsErrors=true` is already set in `Directory.Build.props` — the build step will fail on any warning without needing extra flags. `ubuntu-latest` runners include Docker, so IdentityService.Tests Testcontainers tests (Postgres, RabbitMQ) pull images automatically.

### Task 4: Create `docker.yml` and `dependabot.yml`

**Files:**
- Create: `.github/workflows/docker.yml`
- Create: `.github/dependabot.yml`

**Interfaces:**
- Consumes: Dockerfiles from Tasks 1 and 2
- Produces: Workflow that pushes images to ghcr.io on merge to main; Dependabot config for weekly updates

- [ ] **Step 1: Create `.github/workflows/docker.yml`**

```yaml
name: Docker

on:
  push:
    branches: [main]

permissions:
  contents: read
  packages: write

jobs:
  changes:
    name: Detect changes
    runs-on: ubuntu-latest
    outputs:
      shared: ${{ steps.filter.outputs.shared }}
      api-gateway: ${{ steps.filter.outputs.api-gateway }}
      card-catalog: ${{ steps.filter.outputs.card-catalog }}
      identity-service: ${{ steps.filter.outputs.identity-service }}
      marketplace: ${{ steps.filter.outputs.marketplace }}
      notification: ${{ steps.filter.outputs.notification }}
      portfolio: ${{ steps.filter.outputs.portfolio }}
      pricing: ${{ steps.filter.outputs.pricing }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Detect changed paths
        uses: dorny/paths-filter@v3
        id: filter
        with:
          filters: |
            shared:
              - 'src/SharedKernel/**'
              - 'Directory.Build.props'
              - 'Directory.Packages.props'
            api-gateway:
              - 'src/ApiGateway/**'
            card-catalog:
              - 'src/CardCatalog/**'
            identity-service:
              - 'src/IdentityService/**'
            marketplace:
              - 'src/Marketplace/**'
            notification:
              - 'src/Notification/**'
            portfolio:
              - 'src/Portfolio/**'
            pricing:
              - 'src/Pricing/**'

  build:
    name: Build & push ${{ matrix.service.name }}
    runs-on: ubuntu-latest
    needs: [changes]
    if: needs.changes.outputs.shared == 'true' || needs.changes.outputs[matrix.service.name] == 'true'
    strategy:
      fail-fast: false
      matrix:
        service:
          - name: api-gateway
            project: ApiGateway
            dll: TCGTrading.ApiGateway.dll
          - name: card-catalog
            project: CardCatalog
            dll: TCGTrading.CardCatalog.Api.dll
          - name: identity-service
            project: IdentityService
            dll: TCGTrading.IdentityService.dll
          - name: marketplace
            project: Marketplace
            dll: TCGTrading.Marketplace.Api.dll
          - name: notification
            project: Notification
            dll: TCGTrading.Notification.Api.dll
          - name: portfolio
            project: Portfolio
            dll: TCGTrading.Portfolio.Api.dll
          - name: pricing
            project: Pricing
            dll: TCGTrading.Pricing.Api.dll

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Log in to ghcr.io
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract image metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository_owner }}/tcgtrading-${{ matrix.service.name }}
          tags: |
            type=sha,prefix=sha-,format=short
            type=ref,event=branch
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          file: src/${{ matrix.service.project }}/Dockerfile
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```
